- [x] `[자동]` 구매자가 '구매 확정'을 눌러야만 판매자 지갑으로 정산된다
- [x] `[자동]` 잔액보다 큰 금액은 구매할 수 없다
- [x] `[자동]` 에스크로 진행 중인 상품은 다른 유저가 중복 구매할 수 없다
- [x] `[자동]` 구매자 본인만 확정할 수 있다 (타인 확정 시도 차단)
- [x] `[자동]` 거래 취소 시 구매자에게 전액 환불되고 상품이 판매중으로 복귀한다
- [ ] `[수동]` 지갑 페이지에 잔액과 "에스크로 보관 중" 금액이 구분 표시된다

## 2. 사용자 인증

- [x] `[자동]` 미인증 유저는 상품 등록(판매)이 차단된다
- [x] `[자동]` 미인증 유저는 구매가 차단된다
- [x] `[자동]` 휴대폰 번호는 계정당 하나만 연결된다 (다른 계정에 연결된 번호 거부)

- [x] `[자동]` 아이디/비밀번호 형식 규칙(영문·숫자 4~20자 / 8자 이상 영문+숫자)이 강제된다
